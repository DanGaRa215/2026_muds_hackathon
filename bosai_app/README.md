# BosaiApp

## アプリ概要

東京都江戸川区を対象とした、オフラインファーストの総合防災アプリ MVP。

災害時にはネットワークが断絶する可能性が高いことを前提に設計しており、地震発生から避難完了までのフローを**端末内で完結**できる。平常時には AI による家具の転倒リスク診断や避難ルートの事前計算を行い、発災時には緊急地震速報の受信 → 状況確認 → 避難所推薦 → ナビゲーションまでをシームレスに案内する。

### 解決する課題

- 災害時のネットワーク断絶により避難経路が確認できない
- 家具の転倒防止対策が十分か判断できない
- 災害種別（地震・水害）に応じた最適な避難所が分からない

## 機能一覧

| 機能 | 概要 |
|------|------|
| 緊急地震速報（EEW） | P2P地震情報 WebSocket API によるリアルタイム受信・警告表示 |
| AI 家具安全診断 | 写真から家具の転倒リスクと防止器具の設置状況を AI で判定 |
| オフライン避難ナビ | 事前計算済みルートとオフライン地図による避難経路表示 |
| 避難所推薦 | 距離・標高・災害種別に基づくスコアリングで最適な避難所を提案 |
| 災害モード切替 | 地震モード / 水害モード（洪水・津波・高潮）で避難戦略を変更 |
| 自宅情報登録 | 住所・建物構造の登録と避難ルート事前計算 |
| 診断履歴管理 | 過去の家具診断結果をローカルに保存・閲覧 |

## 機能詳細

### 緊急地震速報（EEW）リアルタイム受信

- P2P地震情報 WebSocket API に常時接続し、緊急地震速報（コード556）・震度情報（551）・津波予報（552）を受信
- 東京都・震度1以上でフィルタリングし、該当する速報を即座に通知
- 受信後は10秒カウントダウン付きの警告画面を表示し、状況確認 → 避難所推薦 → ナビゲーションへ自動遷移
- 指数バックオフ（1秒 → 最大60秒）による自動再接続、ID ベースの重複排除

### AI 家具安全診断

- カメラまたはギャラリーから家具の写真を撮影し、FastAPI バックエンドの画像認識モデルで分析
- 検出項目:
  - 家具の種類（本棚・テレビ・タンス等）
  - 転倒防止器具（L字金具・突っ張り棒・耐震マット等）の有無と設置品質
- リスクレベルを3段階（**危険** / **注意** / **おおむね安全**）で判定し、改善提案を表示
- 診断結果はローカル SQLite に保存し、履歴画面から確認可能

### オフライン避難ナビゲーション

- OpenStreetMap の道路ネットワークを事前コンパイルした `routing.db` を端末に搭載
- A*（単一目標）/ ダイクストラ（複数目標）による経路探索を Dart Isolate で非同期実行
- 3つのルートプロファイル:
  - **ルート1（最速）**: 距離を最小化
  - **ルート2（バランス型）**: 距離と安全性を両立
  - **ルート3（安全重視）**: 安全スコアを最大化し危険箇所を回避
- 自宅登録時に 2モード × 3プロファイル = 6ルートを事前計算してキャッシュ
- PMTiles 形式の東京23区オフライン地図を搭載

### 災害モード別対応

- **地震モード**: 建物倒壊・火災リスクを考慮した避難所選定とルーティング
- **水害モード（洪水・津波・高潮）**: 標高・海岸距離を重視した避難所推薦。適切な避難所がない場合は垂直避難（3階以上）を案内
- 状況確認画面でのチェック内容（けが人・火災・倒壊・津波リスク）に応じて自動判定

### 避難所推薦

- ハバーシン距離・標高・海岸距離・収容人数・対応災害種別に基づくスコアリング
- カード形式のカルーセル UI で候補を順に表示し、ユーザーが選択

### 状態管理

- `StatefulWidget` + `setState()` による画面ローカル状態管理
- `EewManager` / `DatabaseHelper` / `RoutingBootstrap` はシングルトンパターンで共有
- `StreamController` による EEW ログのリアルタイムストリーミング
- `FutureBuilder` による非同期データの読み込み

## 技術スタック

| レイヤー | 技術 |

## セットアップ

```bash
flutter pub get
flutter run
```

### API 設定

家具診断 API の接続先は `--dart-define` で指定する。

```bash
flutter run \
  --dart-define=API_BASE_URL=http://localhost:8000 \
  --dart-define=APP_KEY=your-api-key
```

## オフライン対応

| リソース | 説明 |
|---------|------|
| `assets/routing.db` | OSM 道路ネットワーク + 避難所データ（事前コンパイル済み） |
| `assets/tokyo23_buffered.pmtiles` | 東京23区のオフライン地図タイル |
| SQLite (端末内) | 診断履歴・自宅情報・事前計算済みルート |

ネットワーク不通時でも、避難ルート表示・地図閲覧・診断履歴確認が可能。
